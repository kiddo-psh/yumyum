# /nyam/status 404 — 클래스 레벨 @RequestMapping prefix 충돌
**Date:** 2026-06-23
**Time spent:** ~90분

## Symptom
Vue MyView에서 `GET /nyam/status` 호출 시 404 반환.
Spring Boot 컨트롤러에 `@GetMapping("/nyam/status")`가 명확히 선언되어 있었음에도 응답 없음.

## Wrong tracks

### 1차 시도 — JSON 직렬화 문제 의심 (60분 소비)
- `NyamMood` enum이 JSON으로 직렬화될 때 값이 잘못 매핑되는 게 아닌지 의심
- `@JsonValue`, `@JsonProperty` 어노테이션 추가·제거 반복
- `NyamStatusResponse` DTO 구조 변경 시도
- Spring의 `HttpMessageConverter` 설정 점검
- **결과:** 직렬화와 무관. 요청 자체가 컨트롤러에 도달하지 않고 있었음 (404는 라우팅 실패)

> 교훈: 404는 "컨트롤러가 처리했지만 응답이 잘못됨"이 아니라 "컨트롤러 자체를 찾지 못함"이다.
> 404 발생 시 직렬화보다 **URL 매핑부터** 확인해야 한다.

## Root cause
컨트롤러 클래스에 `@RequestMapping("/growth")`가 클래스 레벨로 선언되어 있었음.

```java
@RestController
@RequestMapping("/growth")   // ← 이게 문제
public class NyamController {

    @GetMapping("/nyam/status")   // 실제 경로: /growth/nyam/status
    public ResponseEntity<NyamStatusResponse> getNyamStatus(...) { ... }
}
```

Vue에서는 `/nyam/status`로 호출 → Spring은 `/growth/nyam/status`에 매핑 → 경로 불일치 → 404.

## Fix
클래스 레벨 `@RequestMapping("/growth")` 제거, 메서드에 전체 경로 직접 명시.

```java
@RestController
// @RequestMapping("/growth") 제거
public class NyamController {

    @GetMapping("/nyam/status")   // 실제 경로: /nyam/status ✅
    public ResponseEntity<NyamStatusResponse> getNyamStatus(...) { ... }
}
```

## Prevention

### 404 디버깅 체크리스트 (우선순위 순)
1. **클래스 레벨 `@RequestMapping` 확인** — 메서드 경로와 합산한 실제 URL을 먼저 계산
2. Spring Boot 기동 로그에서 `Mapped "..."` 라인으로 등록된 실제 경로 확인
3. Nginx 프록시 경로 확인 (nginx.conf의 location 블록)
4. Security Config의 `permitAll()` / `authenticated()` 경로 패턴 확인
5. 그 이후에 직렬화·DTO 문제 의심

### 설계 원칙 (ADR-2로 기록됨)
- 한 컨트롤러에 다른 도메인 경로(`/growth`, `/nyam`)가 혼재하지 않도록 한다.
- 클래스 레벨 prefix를 사용할 경우, 해당 컨트롤러의 모든 경로가 동일 prefix를 공유해야 한다.
- 확실하지 않으면 전체 경로를 메서드에 직접 명시한다.
